More Python Common Mistakes

Additional frequent errors beginners make in Python

Type Mixing in Operations
print(5 + "5")
print(5 + int("5")) # or print(str(5) + "5")
Python doesn't automatically convert between types. You must explicitly convert strings to numbers (with int() or float()) or numbers to strings (with str()) for operations.
Unclosed String Literal
print("Hello)
print("Hello")
Every opening quote needs a matching closing quote of the same type (single or double). Python will give a "SyntaxError: EOL while scanning string literal".
String vs Math Expression
print('5+5') # Outputs '5+5' as text
print(5+5) # Outputs 10 as calculation
Quotes make everything inside them a string literal. For math operations, don't use quotes around the expression.
Boolean Case Sensitivity
print(true)
print(True)
Python's boolean values are case-sensitive keywords. True and False must be capitalized, unlike some other languages.
Boolean vs String
print("True") # This is a string
print(True) # This is a boolean
The quoted version is just text, while the unquoted version is the actual boolean value. This matters for logical operations and conditionals.
Missing Operator
print(5 5)
print(5 * 5) # Or +, -, /
Python requires explicit operators between values. Two numbers next to each other without an operator is a syntax error.
Incomplete Expression
print(type(5+))
print(type(5+3))
The + operator is binary - it needs values on both sides. Python can't determine the type of an incomplete expression.
Division by Zero
print(5 / 0)
print(5 / 0.1) # Or any non-zero value
Division by zero is mathematically undefined and raises a ZeroDivisionError in Python. Always check denominators before dividing.

Remember: Understanding these mistakes will help you debug code faster!
