Common Python Mistakes

Learn to identify and fix frequent errors beginners make

Missing Quotes
print(Hello)
print("Hello")
Strings in Python must be enclosed in quotes. You can use single (' ') or double (" ") quotes, but they must match.
Missing Parentheses
print"Hello"
print("Hello")
In Python 3, print is a function and requires parentheses. This was different in Python 2.
Mixed Quote Types
print("Hello')
print("Hello") or print('Hello')
Don't mix single and double quotes to start and end a string. Choose one style and be consistent.
Case Sensitivity
Print("Hello")
print("Hello")
Python is case-sensitive. Print with capital P is not the same as print.
Incomplete Math Operation
print(5 + )
print(5 + 3)
All math operations need complete expressions with both operands. The plus operator needs a number on both sides.
Incomplete Second Argument
print("Hello", 5 + )
print("Hello", 5 + 3)
When printing multiple items, each argument must be complete. The incomplete math operation causes a syntax error.
Forgot Closing Quote
print("Hello)
print("Hello")
Every opening quote needs a matching closing quote. Python will think your string continues to the next line.
Missing Closing Parentheses
print(10, 20, 30
print(10, 20, 30)
Every opening parenthesis must have a matching closing one. Count your parentheses to make sure they match.
Missing Quotes Around Words
print("Coding", is, "fun")
print("Coding", "is", "fun")
Text strings must be in quotes. Without quotes, Python thinks is is a variable name.
Emoji Without Quotes
print( )
print("")
Emojis are text characters and must be enclosed in quotes like any other string.

Remember: Everyone makes mistakes - they're opportunities to learn!
