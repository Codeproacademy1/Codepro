Python Variable Mistakes

Common errors when working with variables in Python

Unclosed String in Variable Assignment
x = "Sara
x = "Sara"
String literals must be enclosed in matching quotes. Missing the closing quote will result in a SyntaxError: EOL while scanning string literal.
Invalid Variable Name Starting with Number
2name = "Ali"
name2 = "Ali" # or _2name, name_2
Variable names cannot start with numbers in Python. They must start with a letter or underscore, followed by letters, numbers, or underscores.
Using Undefined Variable
print(name)
name = "John" # Define first print(name)
Python raises a NameError when you try to use a variable that hasn't been defined yet. Always assign values to variables before using them.
Missing Comma Between Print Arguments
print("Hello" name)
print("Hello", name)
When printing multiple items, they must be separated by commas. Without the comma, Python thinks you're trying to concatenate strings (which would need a + operator).
Dynamic Typing Confusion
x = 5 x = "hello" # Now x + 5 causes error
x = 5 x = str(x) + "hello" # Explicit conversion
Python is dynamically typed, so variables can change type. But operations between incompatible types (like number + string) still need explicit conversion.
Case-Sensitive Function Name
Print(x)
print(x)
Python is case-sensitive. The built-in function is print() with lowercase 'p'. Print() would raise a NameError unless you defined it yourself.
Spaces in Variable Names
my variable = 10
my_variable = 10 # or myVariable
Variable names cannot contain spaces. Use underscores (_) for word separation (snake_case) or camelCase for multi-word variables.
Incomplete Variable Assignment
x =
x = None # or some default value
An equals sign must have both a variable name and a value. If you need a placeholder, use None or some default value.
Forgetting to Print Function Results
type(5) # Output disappears
print(type(5)) # Shows <class 'int'>
In interactive mode, Python shows results automatically, but in scripts you need print() to see output. type() returns a value that won't be shown without printing.

Tip: Pay attention to error messages - they often tell you exactly what's wrong!
