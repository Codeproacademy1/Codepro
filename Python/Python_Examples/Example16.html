1 Single-Line Comment 
# This is a single-line comment
print("Hello")  # This comment is after code
Output: Hello

Use # for single-line comments. Everything after # is ignored.

2 Multi-Line Comment (Docstring) 
"""This is a multi-line comment
It can span multiple lines
Python will ignore this too"""
print("World")
Output: World

Triple quotes (""" or ''') create multi-line comments.

3 Commenting Out Code 
# print("This won't run")
print("This will run")
Output: This will run

Comments can disable code without deleting it - great for testing!

4 Function Documentation 
def greet(name):
    """This function greets the person passed in"""
    print(f"Hello, {name}!")

greet("Alice")
Output: Hello, Alice!

Docstrings document what functions do (appears right after definition).

5 TODO Comments 
# TODO: Add error handling here
print("Processing data...")
Output: Processing data...

TODO comments remind you about work needed later.

6 Commented Math Example 
# Calculate rectangle area
length = 5  # in centimeters
width = 3   # in centimeters
area = length * width
print(f"Area: {area} cm¬≤")
Output: Area: 15 cm¬≤

Comments explain what variables mean and what the code does.

7 Module-Level Docstring 
"""weather.py - Weather forecasting module

This module provides weather forecasts
for different locations."""

def get_forecast(city):
    # Implementation would go here
    return "Sunny"
Note: This is module documentation, not executable code

Module docstrings explain what the entire file does (at the top).

8 Commented Conditional 
age = 15

# Check if person can vote
if age >= 18:
    print("You can vote!")
else:
    # Show message for underage
    print("Come back when you're 18")
Output: Come back when you're 18

Comments explain the logic behind conditionals.

9 Commented Loop 
# Print numbers 1 to 5
for i in range(1, 6):
    # Skip number 3
    if i == 3:
        continue
    print(i)
Output:
1
2
4
5

Comments clarify loop behavior and special cases.

10 Comment Style Guide 
# Good comment (explains why)
x = x + 1  # Compensate for border

# Bad comment (repeats what code says)
x = x + 1  # Increment x by 1

"""
Good docstring format:
- First line summary
- Blank line
- Detailed description
"""
Tip: Focus on explaining why, not what

Follow these commenting best practices for clean code.
